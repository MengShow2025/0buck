## 1. 产品概述

| 字段 | 内容 |
|------|------|
| **产品名称** | 0buck |
| **产品定位** | 基于AI多模态交互与1688/阿里巴巴全自动一件代发的跨境电商平台，让海外买家通过自然对话即可完成购物全流程 |
| **核心价值主张** | "The store that listens" — 用AI替代传统搜索框，用对话替代分类浏览，用自动化供应链替代人工运营 |
| **目标用户** | 美国、欧洲、东南亚个人买家 |
| **商品范围** | 小件快消品（家居、户外、母婴、配件等） |
| **商业模式** | 差价盈利（从1688/阿里拉货定价后上架） |
| **底层技术框架** | **Shopify (Headless Commerce) + 自定义AI编排后端** |

### 5.2 页面详细需求

#### 页面1：AI对话窗口（Agentic Chat — 核心页面）

-   **页面目标**：作为用户与平台交互的唯一主入口，替代传统搜索框和导航栏
-   **核心功能**：
    -   多模态输入支持（文字、图片、视频链接、第三方商品链接）
    -   实时意图识别，路由到对应AI代理
    -   商品推荐卡片流式展示
    -   多轮对话上下文记忆
    -   客服问题实时解答
    -   订单状态内嵌查询
-   **业务逻辑**：
    -   用户发送图片 → 视觉特征提取 → 向量库语义匹配 → 返回Top3商品卡片
    -   用户描述需求 → LLM意图解析 → 子需求拆解 → 多品类匹配
    -   用户询问规则/活动 → 知识库RAG检索 → 结构化回答
    -   用户查询订单 → **调用自定义后端订单代理 (Shopify Storefront API查询基础信息，自定义后端处理订单履行状态)** → 格式化物流信息返回
    -   置信度低于阈值 → 升级到人工客服
-   **页面元素**：
    -   顶部：对话历史记录
    -   中部：商品卡片流（图片、价格、匹配理由、操作按钮）
    -   底部：输入框（支持"+"添加图片/视频）、发送按钮
    -   状态提示："正在为您匹配商品... Thought for 1.2s"
-   **交互逻辑**：
    -   商品卡片点击 → 底部滑出H5预览 / WhatsApp跳转H5链接
    -   "查看更多" → 展开商品画布页
    -   "直接购买" → **生成 Shopify Checkout URL 并引导用户完成结账**
    -   长按卡片 → 收藏商品 (通过自定义后端记录)

---

#### 页面2：H5商品详情页（Product Detail Page）

-   **页面目标**：承接AI卡片点击，提供沉浸式商品浏览与购买决策支持
-   **核心功能**：
    -   商品高清图集（**通过 Shopify Storefront API 获取，并结合 Metafields 存储的 1688 源图** + AI翻译描述）
    -   SKU选择器（颜色、尺码、规格）
    -   实时运费+税费计算器（基于收货地址）
    -   供应商资质展示（CE/ISO/FDA认证）
    -   用户评价**从 1688 同步翻译到自定义后端并展示**）
    -   一键加入购物车 / 直接购买
-   **业务逻辑**：
    -   页面数据来源：**Shopify Storefront API (产品核心数据) + Shopify Metafields (1688原始数据、供应商关联ID) + 自定义后端 (AI翻译处理、用户评价)**
    -   运费实时计算：基于SKU重量体积 + 目的地 + 物流商API（**由自定义后端或前端调用 ShippyPro 等服务实现**）
    -   税费计算：集成Avalara/TaxCloud，根据目标国VAT自动计算（**由自定义后端或前端调用服务实现**）
    -   库存实时同步：**通过 Shopify Storefront API 获取实时库存**，低库存由 Shopify 管理。
-   **页面元素**：
    -   顶部：商品图片轮播
    -   中部：商品名称（AI翻译）、价格（含差价）、SKU选择器
    -   信任区：供应商评分、认证徽章、退换货政策摘要
    -   到岸成本区：运费+关税+VAT实时展示
    -   底部：固定"加入购物车"/"立即购买"按钮
-   **交互逻辑**：
    -   在飞书/WhatsApp内以侧边栏/内嵌浏览器打开（保持对话窗口可见）
    -   用户行为实时回传AI代理（浏览深度、点击SKU、停留时间）→ 优化向量推荐

---

#### 页面3：结账支付页（Checkout）

-   **页面目标**：完成安全支付，收集配送信息，触发1688自动下单
-   **核心功能**：
    -   收货地址填写（支持地址自动补全）
    -   物流方式选择（标准/快递/经济）
    -   优惠券/活动码输入
    -   **Shopify Payments 集成** 或 **Shopify Checkout UI 扩展**
    -   订单确认邮件/WhatsApp消息自动发送
-   **业务逻辑**：
    -   用户提交订单 → **Shopify 创建 Order** → **Shopify Webhook 触发自定义后端 (AI订单代理)** 进行 1688 自动采购
    -   1688采购成功 → **自定义后端通过 Shopify Admin API 更新 Shopify 订单状态** → 推送用户确认消息
    -   支付失败 → AI主动询问是否需要帮助
-   **交互逻辑**：
    -   付款成功 → 跳转订单追踪页 + WhatsApp发送确认卡片

---

#### 页面4：订单追踪页（Order Tracking）

-   **页面目标**：全程透明展示订单状态，降低用户焦虑
-   **核心功能**：
    -   订单状态时间线（下单→采购→发货→在途→已签收）
    -   物流轨迹地图（可选）
    -   预计送达时间
    -   问题反馈入口（直接唤起AI对话）
-   **业务逻辑**：
    -   物流数据：**通过自定义后端调用 ShippyPro API 实时拉取，并更新 Shopify Order Fulfillment 状态**
    -   延误检测：AI监控时效，超时主动推送WhatsApp预警 + 补偿优惠券
    -   异常订单：自动升级人工处理

## 6. 用户故事

### P0 核心功能（MVP）

**找货场景：**
-   作为海外买家，我希望发一张图片给AI就能找到同款商品，以便省去繁琐的搜索过程
    -   业务规则：图片匹配后返回Top3商品，响应时间 < 3秒
-   作为海外买家，我希望用自然语言描述需求（如"给孩子的环保餐具"），AI能准确理解并推荐，以便不需要知道专业搜索词
    -   业务规则：意图识别准确率 > 90%，支持英语/西班牙语/泰语/越南语

**购买场景：**
-   作为海外买家，我希望在详情页看到最终到岸价（含运费+税），以便知道实际花费
    -   业务规则：运费+VAT实时计算，误差 < 5%
-   作为海外买家，我希望下单后自动发货，不需要等待人工处理，以便体验快速发货
    -   业务规则：**Shopify 订单支付成功后 < 2小时自动触发自定义后端进行 1688 采购**

**客服场景：**
-   作为海外买家，我希望随时通过AI查询我的订单状态，以便不需要联系人工客服
    -   业务规则：AI查询物流响应 < 2秒，准确率 = 100%
-   作为海外买家，我希望AI能解答退换货政策以��购买前打消顾虑
    -   业务规则：知识库覆盖率 > 95%的常见问题，无法回答时转人工

### P1 重要功能

**推荐场景：**
-   作为海外买家，我希望AI根据我的历史偏好主动推送感兴趣的商品，以便发现好物不需要主动搜索
    -   业务规则：基于用户样式护照，每日推送 ≤ 3次，可关闭

**活动场景：**
-   作为海外买家，我希望AI主动告知我有什么可用的优惠活动，以便减少花费
    -   业务规则：活动信息从 **Shopify 促销模块** 实时同步，优惠券自动可用

## 7. AI代理功能规范

### 7.1 意图识别与路由

| 用户输入类型 | 意图识别 | 路由代理 | 响应形式 |
|:------------:|:--------:|:--------:|:--------:|
| 图片/视频/链接 | 找货意图 | 检索代理（Retrieval Agent） | 商品卡片 × 3 |
| 自然语言描述需求 | 找货意图 | 检索代理 | 商品卡片 × 3 |
| 订单号/订单查询 | 订单意图 | 订单代理（Order Agent） | 物流状态卡片 |
| 退换货/投诉 | 售后意图 | 客服代理（CS Agent） | 结构化文字+操作按钮 |
| 政策/规则咨询 | 知识意图 | 知识库代理（KB Agent） | 结构化文字回答 |
| 活动/优惠咨询 | 活动意图 | 活动代理（Promo Agent） | 活动卡片+优惠码 |
| 模糊/闲聊 | 引导意图 | 引导代理 | 引导用户明确需求 |

### 7.2 AI客服知识库模块

**知识库分类：**

| 分类 | 内容 | 更新方式 |
|:----:|:----:|:--------:|
| 退换货政策 | 7天无理由、商品损坏赔付、退款时效 | 人工维护 |
| 物流政策 | 各国时效、运费计算规则、禁运品清单 | 人工维护 |
| 支付政策 | 支持货币、手续费、支付安全 | 人工维护 |
| 活动规则 | 当前促销、使用门槛、叠加限制 | **Shopify 促销模块同步** |
| 商品咨询 | 材质、认证、使用场景 | 1688数据AI提取 |
| 清关合规 | 各国进口规定、税率表、申报要求 | 人工维护 + 定期更新 |

**知识库技术实现：**
-   所有知识库文档向量化存储至Qdrant
-   用户提问 → RAG检索最相关文档 → LLM生成结构化回答
-   置信度 < 0.7 → 自动追加"联系人工客服"选项

### 7.3 多轮对话管理

```
会话开始
  ↓
用户输入（文字/图片/视频/链接）
  ↓
意图识别（置信度评分）
  ↓
路由至对应代理
  ↓
向量检索 / API调用 / 知识库查询
  ↓
生成响应（商品卡片/文字/表单）
  ↓
用户行为反馈（点击/购买/收藏）
  ↓
更新用户样式护照 → 优化下次推荐
  ↓
下一轮输入（保持上下文）
```

**上下文记忆规则：**
-   单次会话内：完整对话历史保留
-   跨会话：保留用户偏好标签（样式护照）、历史订单、收货地址
-   隐私保护：PII数据（姓名、地址、联系方式）加密存储，不传递给LLM

## 8. 供应链自动化规范

### 8.1 商品上架流程

```
1688商品数据抓取（ElimAPI/DSers API）
  ↓
AI属性提取与翻译（中文→英文/多语言）
  ↓
多模态向量化（SigLIP图文联合嵌入）
  ↓
存入Qdrant向量库
  ↓
差价定价策略应用（规则引擎 - 自定义后端）
  ↓
**通过 Shopify Admin API 上架至 Shopify (Metafields 存储原始信息)**
  ↓
定期库存价格同步（每4小时，通过 Shopify Admin API）
```

### 8.2 订单履约自动化流程

```
用户下单（自定义前端触发 Shopify Checkout）
  ↓
支付成功（Shopify Payments 回调）
  ↓
**Shopify Webhook 触发自定义后端 (AI订单代理)**
  ↓
自定义后端 (AI订单代理) 调用DSers/1688 API自动采购
  ↓
1688确认发货 → 获取运单号
  ↓
**自定义后端通过 Shopify Admin API 更新 Shopify 订单状态和履行信息**
  ↓
WhatsApp/邮件推送发货通知
  ↓
物流轨迹实时同步（ShippyPro API → 自定义后端 → Shopify Admin API）
  ↓
用户签收 → 自动完成订单
  ↓
满意度调查推送（3天后）
```

### 8.3 差价定价策略

## 9. 支付与合规

### 9.1 支付架构

| 渠道 | 用途 | 优先级 |
|:----:|:----:|:------:|
| **Shopify Payments** | Web端主要支付 | 主选 |
| WorldFirst（跨境宝） | 1688供应商结算 | 主选 |
| PayPal | 备用支付选项 | 备选 |
| Stripe | 东南亚信用卡支付 | P1阶段 |

## 14. 产品约束

-   **平台要求**：MVP仅覆盖WhatsApp + Web端，暂不开发独立App
-   **功能边界**：暂无供应商入驻功能，所有货源来自1688/阿里
-   **AI边界**：AI不参与价格谈判，不执行支付操作，仅辅助决策
-   **支付边界**：用户支付完全托管于 Shopify Payments，平台不存储支付卡信息。对供应商支付通过 WorldFirst 等由自定义后端编排。
-   **数据边界**：用户PII数据加密存储，不传递给第三方LLM服务
-   **内容边界**：禁运品（危险品/仿品/违禁品）自动过滤，不上架
