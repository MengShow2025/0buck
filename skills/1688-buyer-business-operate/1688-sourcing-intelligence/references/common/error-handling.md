# 通用错误处理

所有命令在遇到以下 HTTP / 业务错误时，遵循统一处理策略。

## 错误码与 Agent 应对

| 错误码 | 含义 | Agent 应对 |
|-------|------|-----------|
| — | AK 未配置 | 输出 AK 引导话术 |
| 400 | 参数不合法 | 检查用户输入是否正确 |
| 401 | 鉴权无效 | 输出 AK 引导话术，检查 AK/SK |
| 429 | 请求被限流 | 建议等待 1-2 分钟后重试 |
| 500 | 服务端异常 | 建议稍后重试 |

## 网络异常

CLI 已内置 3 次重试（指数退避），重试耗尽后返回 success: false。

Agent 应对：告知用户"网络异常，请检查网络连接后重试"。

## 识别方式

当 CLI 输出 success: false 时：

1. 输出 markdown 字段（用户可读的错误描述）
2. 检查 markdown 中的关键词，按 SKILL.md 异常处理表追加对应引导

## 关键词与引导

| 关键词 | Agent 额外动作 |
|-------|---------------|
| AK 未配置 | 引导用户配置 AK |
| 签名无效 | 引导用户检查 AK/SK |
| 限流 | 建议等待重试 |
| 网络异常 | 建议检查网络 |
| 超时 | 建议稍后重试 |

## 各能力特有异常

见各能力文档的"异常处理"段。
