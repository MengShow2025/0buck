# 0Buck 奖励与营销逻辑详细执行规范 (v1.0)

本规范为 **程序猿** (后端实现) 和 **ai-architect** (AI 交互) 提供具体的业务逻辑细节，确保 0Buck 的“555天签到”和“分销体系”在代码层面无偏差执行。

---

## 1. 签到返现 (Check-in Cashback) 细则

### 1.1 时间与时区规则
- **计算基准**: 采用用户收货地址对应的 **当地时间** (User Local Time)。
- **开启时间**: 订单状态变为 `fulfilled` 且物流显示“已签收”后的 **次日凌晨 00:00** 自动开启第一期。
- **签到窗口**: 每日 00:00 - 23:59。

### 1.2 异常中断处理 (Strict Consecutive Rule)
- **中断定义**: 凡是在当前周期内（如 P1 的 5 天内）漏签任何一天，该期奖励立即失效。
- **自动顺延**: 中断后的下一次签到，系统自动将其标记为 **下一期 (Next Period)** 的第一天。
    - *示例*: 用户在 P1 (需5天) 的第3天漏签，第4天点击签到，此时第4天变为 **P2** 的第1天。
- **损失计算**: 用户失去了 P1 的 5% 返现机会。

### 1.3 555天强制结算 (Final Settlement)
- 若达到 555 天期限，但当前期未完成（如 P20 进行到第15天）：
    - **补偿公式**: `奖励 = (当前期已连续天数 / 30) * 5% * 奖励基数`。
    - 该金额在第 556 天自动发放至钱包。

## 2. 拼团免单 (Group-buy) 互斥逻辑

- **二选一限制**: 用户在确认收货后 5 天内（或第一期签到完成前），可在 App/WhatsApp 界面选择“拼团免单”模式。
- **触发条件**:
    - 分享链接产生的关联订单数 $\ge 3$。
    - 关联订单均已支付且 **未退款**。
- **返现执行**: 一旦满足 3 人条件，该订单的“签到返现”计划自动终止，全额订单金额（实付）返还至钱包。

---

## 3. 推广奖励 (Referral) 等级定义

为落实 PRD 中“根据不同级别 1.5% - 3%”的要求，定义以下普通用户分销等级：

| 等级名称 | 达成条件 (2年内累计受邀交易额) | 奖励比例 |
| :--- | :--- | :--- |
| **Silver (初级)** | $0 - $1,000 | 1.5% |
| **Gold (中级)** | $1,000 - $5,000 | 2.0% |
| **Platinum (高级)** | > $5,000 | 3.0% |

- **结算时机**: 受邀人订单状态变为 `Archived` (签收过退货期) 后，奖励由 `locked` 转为 `available`。

## 4. 退款与逆向流程 (Edge Cases)

- **整单退款**:
    - 立即终止对应的 `checkin_plan`。
    - 扣回该订单已发放的所有签到奖励（若余额不足，钱包显示负数，从下一笔奖励中扣除）。
- **部分退款**:
    - 重新计算 `reward_base` (奖励基数)。
    - 后续期的 5% 按新基数计算，已发放部分不予追讨，但总返现额不得超过新基数。
