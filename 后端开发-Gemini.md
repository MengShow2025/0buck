## 优化后的后端开发计划书

**核心策略：** 以 Saleor 为基础商业引擎，通过其 GraphQL API 和 Webhooks 进行功能扩展和集成；AI 代理层作为核心，采用 LangGraph 框架与 Saleor 协同工作，实现 AI 驱动的交互逻辑。

### Phase 1 — MVP（WhatsApp + 基础找货下单）

**目标：** 跑通“找货 → 详情 → 下单 → 自动发货”的核心闭环。

#### 1. Saleor 基础架构搭建与配置 (商业逻辑层)
*   **部署 Saleor:** 在 AWS/GCP 上部署 Saleor 核心服务（Docker + PostgreSQL + Redis）。
*   **Schema 扩展:** 根据产品需求（如商品额外的 1688 源数据字段、供应商的详细资质信息、用户样式护照等），扩展 Saleor 的 GraphQL Schema 和模型。
*   **模块配置:**
    *   **商品管理 (PIM):** 配置 Saleor 的商品、属性、SKU、多图片管理。
    *   **订单管理 (OMS):** 配置订单生命周期、状态、折扣和税费模块。
    *   **用户管理:** 配置用户账户、地址管理。
    *   **渠道管理:** 配置 Web 和 WhatsApp 渠道，支持独立定价和库存。

#### 2. 数据层建设与同步
*   **Qdrant 向量数据库部署:** 部署 Qdrant 服务。
*   **1688 商品同步流水线 (Python/Django):**
    *   **ElimAPI/DSers API 集成:** 开发服务（可能作为 Saleor 的 Django App 或独立微服务）接入 1688/阿里巴巴 API，定时抓取商品数据。
    *   **AI 属性提取与翻译:** 使用 AI 服务（例如基于 `@google/genai` 或其他 NLP 模型）对抓取的中文商品描述、属性进行多语言（英语为主）翻译和结构化提取。
    *   **多模态向量化:**
        *   使用 **SigLIP** 或类似的多模态嵌入模型，对商品图片和翻译后的文本描述进行联合向量化。
        *   将这些向量连同商品元数据（如 ID、价格、品牌、类别）存入 Qdrant 向量数据库，并与 Saleor 中的商品 ID 关联。
    *   **差价定价策略:** 实现基于 PRD 中定义的差价倍率规则引擎，计算商品的最终售价，并更新至 Saleor。
    *   **上架与同步:** 通过 Saleor 的 GraphQL Product API 将处理后的商品数据（包括价格、库存）上架。定期（例如每 4 小时）同步 1688 的库存和价格变动。

#### 3. AI 代理层核心开发 (基于 LangGraph/Python)
*   **AI 代理服务部署:** 部署一个 Python 服务，作为 AI 代理层。
*   **意图识别引擎:** 开发核心模块，接收用户输入（文字、图片、视频链接、商品链接），利用 LLM 或专门的 NLP 模型进行意图识别（找货、订单查询、客服、活动等），并评估置信度。
*   **多轮对话管理:** 实现对话上下文记忆机制，支持跨会话的用户偏好（样式护照）、历史订单、收货地址的存储和检索（注意 PII 数据加密）。
*   **找货代理 (Retrieval Agent):**
    *   接收用户找货请求。
    *   根据用户输入（图片/描述）生成查询向量。
    *   在 Qdrant 中执行向量检索，匹配 Top N（例如 Top 3）最相似的商品。
    *   从 Saleor 或直接从 Qdrant 的元数据中获取商品详情。
    *   生成包含商品卡片（图片、价格、匹配理由、操作按钮）的响应。

#### 4. 交互层集成 (Next.js H5 & WhatsApp Bot)
*   **H5 商品详情页 (Next.js):**
    *   开发 Next.js 应用，作为 Saleor 的 Headless Storefront。
    *   通过 Saleor 的 GraphQL API 获取商品高清图集、SKU 选项、供应商资质、用户评价。
    *   实现实时运费和税费计算器（集成 ShippyPro API 获取运费，集成 Avalara/TaxCloud 计算税费）。
    *   集成 Saleor Checkout API 实现“一键加入购物车 / 直接购买”。
*   **WhatsApp Business API 集成:**
    *   接入 Meta Business Cloud API。
    *   开发接口接收 WhatsApp 消息，并将用户输入转发给 AI 代理层。
    *   将 AI 代理层生成的响应（商品卡片、文字）格式化为 WhatsApp 交互式消息或 H5 链接返回给用户。

#### 5. 订单履约自动化
*   **支付集成:**
    *   集成 Shopify Payments（Web 端主支付）和 WorldFirst API（用于 1688 供应商结算）。
    *   当用户在 H5 详情页完成支付后，通知 Saleor 更新订单状态。
*   **Saleor Webhook 配置:**
    *   配置 Saleor Webhook，监听“订单支付成功”事件。
    *   当 Webhook 触发时，AI 订单代理（或专门的订单处理服务）接收通知。
*   **自动采购与物流跟踪:**
    *   **DSers/1688 API 自动采购:** AI 订单代理调用 DSers/1688 API 自动完成采购。
    *   **运单号获取:** 1688 确认发货后，获取运单号。
    *   **Saleor 订单状态更新:** 更新 Saleor 订单状态，并记录运单号。
    *   **物流轨迹同步:** 集成 ShippyPro API，实时拉取物流数据，并同步到 Saleor。
    *   **通知发送:** 通过 WhatsApp/邮件推送发货通知和物流更新。

#### 6. 基础非功能性需求
*   **可观测性:** 搭建日志、监控和告警系统。
*   **安全性:** 确保所有 API 密钥、支付凭证的加密存储（例如使用 AWS Secrets Manager），AI 代理无直接支付执行权限，PII 数据加密存储。
*   **性能:** 关注 AI 响应时间 (TTFT < 1秒)、商品匹配端到端延迟 (< 3秒)。

### Phase 2 — AI 增强（Web平台 + 客服知识库）

**目标：** 扩展 Web 平台功能，引入 AI 客服和个性化推荐。

#### 1. Web 平台首页与嵌入式 AI 聊天窗口 (Next.js)
*   **开发 Web 平台首页:** 包含 AI 聊天窗口和 AI 推荐商品流。
*   **AI 对话窗口组件:** 开发可嵌入的 AI Chat 组件，与 AI 代理层交互。
*   **用户样式护照:** 完善用户偏好收集机制，建立用户样式护照。
*   **推荐代理 (Recommendation Agent):** 基于用户样式护照和行为数据（浏览、点击、购买、收藏），通过 Qdrant 进行推荐商品检索，主动推送个性化推荐。

#### 2. AI 客服知识库模块
*   **知识库文档向量化:** 将 PRD 中定义的知识库文档（退换货、物流、支付政策等）向量化后存储至 Qdrant。
*   **知识库代理 (KB Agent):**
    *   接收客服问题意图。
    *   利用 RAG (Retrieval Augmented Generation) 机制，在 Qdrant 中检索最相关文档。
    *   通过 LLM 生成结构化回答。
    *   实现置信度判断，低于阈值时提供“联系人工客服”选项。
*   **售后意图代理 (CS Agent):** 处理退换货、投诉等售后意图，与 KB Agent 协同工作，并可能集成工单系统。

#### 3. 活动代理 (Promo Agent)
*   集成 Saleor 促销模块，同步优惠券、限时活动信息。
*   AI 活动代理根据用户情境或查询，主动推送活动卡片和优惠码。

#### 4. 多语言支持
*   扩展 AI 代理层的意图识别和响应生成能力，支持西班牙语/越南语/泰语等。

### Phase 3 — 生态扩展

**目标：** 进一步扩展供应链和平台功能。

*   **扩展供应链集成:** 接入 Shopify 供应商、Spocket 等其他供应链平台。
*   **视频找货:** 开发视频内容解析能力，提取商品特征进行向量化，实现视频找货。
*   **用户评价系统:** 实现用户评价功能，并利用 AI 摘要用户评价。
*   **会员体系:** 在 Saleor 中实现积分、等级、专属优惠等会员功能。
